Veritabanı Şeması

Performans Analiz Sistemi'nin detaylı veritabanı tasarımı ve ilişkileri

Ana Veritabanı Tabloları

Users

Sistem kullanıcıları ve çalışan bilgileri
  • Id INT (PK)
  • EmployeeId VARCHAR(50)
  • FirstName NVARCHAR(100)
  • LastName NVARCHAR(100)
  • Email VARCHAR(255)
  • DepartmentId INT (FK)
  • ManagerId INT (FK)
  • IsActive BIT
  • CreatedAt DATETIME2
  • LastLoginAt DATETIME2

ActivityLogs

Kullanıcı aktivite kayıtları
  • Id BIGINT (PK)
  • UserId INT (FK)
  • WindowTitle NVARCHAR(500)
  • ApplicationName NVARCHAR(200)
  • ProcessName NVARCHAR(200)
  • StartTime DATETIME2
  • EndTime DATETIME2
  • Duration INT
  • CategoryId INT (FK)
  • IsProductive BIT

Categories

Aktivite kategorileri ve alt kategoriler
  • Id INT (PK)
  • Name NVARCHAR(100)
  • Description NVARCHAR(500)
  • ParentId INT (FK)
  • Level INT
  • IsProductive BIT
  • Color VARCHAR(7)
  • IsActive BIT

CategoryKeywords

Kategori eşleştirme anahtar kelimeleri
  • Id INT (PK)
  • CategoryId INT (FK)
  • Keyword NVARCHAR(200)
  • KeywordType VARCHAR(20)
  • Priority INT
  • IsActive BIT

Departments

Organizasyon departmanları
  • Id INT (PK)
  • Name NVARCHAR(100)
  • Code VARCHAR(20)
  • ManagerId INT (FK)
  • IsActive BIT

Reports

Oluşturulan raporlar
  • Id INT (PK)
  • Name NVARCHAR(200)
  • ReportType VARCHAR(50)
  • UserId INT (FK)
  • StartDate DATE
  • EndDate DATE
  • GeneratedAt DATETIME2
  • Status VARCHAR(20)

Veri Akış Şeması

1

Veri Toplama

Client ajanı pencere başlıkları ve uygulama bilgilerini toplar

2

Yerel Depolama

Veriler şifrelenerek yerel SQLite veritabanında saklanır

3

Sunucu Transferi

HTTPS ile güvenli şekilde sunucuya gönderilir

4

Veri İşleme

Kategorizasyon ve analiz işlemleri yapılır

5

Veritabanı Kaydı

İşlenmiş veriler SQL Server'a kaydedilir

6

Raporlama

Dashboard ve raporlar gerçek zamanlı oluşturulur

İndeksler ve Performans Optimizasyonu

Önerilen İndeksler

  • ActivityLogs: UserId, StartTime, EndTime (Composite Index)
  • ActivityLogs: CategoryId, StartTime (Composite Index)
  • ActivityLogs: ApplicationName (Non-clustered Index)
  • Users: EmployeeId (Unique Index)
  • Users: Email (Unique Index)
  • Users: DepartmentId (Non-clustered Index)
  • CategoryKeywords: Keyword, KeywordType (Composite Index)
  • Reports: UserId, GeneratedAt (Composite Index)

Veri Güvenliği

  • Şifreleme: Hassas veriler AES-256 ile şifrelenir
  • Yedekleme: Günlük tam yedekleme ve saatlik log yedekleme
  • Veri Saklama: 90 gün sonra otomatik arşivleme
  • Erişim Kontrolü: Row-level security ile kullanıcı bazlı erişim
  • Audit Log: Tüm veri değişiklikleri loglanır

Performans Metrikleri

  • Veri Ekleme: 1000+ kayıt/saniye
  • Sorgu Yanıt Süresi: < 100ms (ortalama)
  • Eşzamanlı Kullanıcı: 1000+ kullanıcı
  • Veri Boyutu: 1TB+ veri desteği
  • Uptime: %99.9 kullanılabilirlik
